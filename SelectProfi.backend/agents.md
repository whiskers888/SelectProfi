# Agent Behavior

## Task Source
- Основной источник задач: YouGile (через MCP)
- Всегда опирайся только на данные из задачи
- Не додумывай требования

## Workflow

Каждая задача выполняется строго по шагам:

1. Прочитать задачу
2. Проанализировать
3. Разбить на шаги
4. Выполнить ОДИН шаг
5. Сформировать комментарий
6. Остановиться

## Step Rules

- Один ответ = один шаг
- Никаких действий вне текущего шага
- Не реализовывать наперёд

## Task Analysis

Перед выполнением:
- Проверить полноту задачи
- Если есть неопределённость → задать вопрос
- Не писать код при неясных требованиях

## YouGile Comments

После каждого шага формируй комментарий:

Структура:

Статус: (in progress / done / blocked)

Что сделано:
- ...

Что дальше:
- ...

Риски:
- ...

## Blocking Rules

Если задача неясна:

- Остановиться
- Установить статус: blocked
- Сформулировать вопрос

## Code Rules (Short)

- Минимальный код
- Без комментариев
- Без any
- Production-ready
- ESLint clean

## Scope Control

Запрещено:
- Делать несколько шагов сразу
- Добавлять лишний функционал
- Изменять несвязанный код

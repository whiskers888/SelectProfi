# Прогресс по `manual-flow-order-vacancy-candidate`

Дата обновления: 2026-04-13

## Статус шагов

- Шаг 1 (подтвержден): кнопка «Вы уже откликнулись» обновляется сразу после отклика (через `refetchMyOrderResponse` после `respond`).
- Шаг 1 (подтвержден): после `logout/login` не остаются данные прошлого аккаунта без `F5` (через `api.util.resetApiState()` на `logout/login/register/demo`).
- Шаг 2 (подтвержден): для исполнителя видимость заказов в дашборде соответствует сценарию (backend возвращает `assigned-to-me + unassigned`, UI корректно отображает эти заказы).
- Шаг 3 (в работе): удален legacy-переход на `/vacancies`; создание вакансии перенесено в `workspace` как отдельная пользовательская панель (по аналогии с созданием заказа/кандидата).
- Шаг 3.1 (выполнен): добавлено автосохранение и восстановление `draft` создания вакансии в `workspace` (персистентно при закрытии формы, ключ `userId + orderId`).
- Шаг 3.2 (выполнен): добавлена отдельная отправка заказчику из формы (`create Draft` + отдельная кнопка `Отправить заказчику` с переводом в `OnApproval`).
- Шаг 3.3 (выполнен): добавлен заметный UI-индикатор незавершенного черновика в деталях заказа и в форме создания вакансии.
- Шаг 3.4 (выполнен): в форме вакансии подключен общий `RichTextEditor` для поля описания с валидацией по видимому тексту.
- Шаг 3.5 (выполнен): добавлено подтверждение вакансии заказчиком в `workspace` (`OnApproval -> Published`) из деталей заказа.
- Шаг 3.6 (выполнен): исправлен `409 vacancy_conflict` при повторной отправке заказчику — для существующей вакансии по заказу используется `update`, а не повторный `create`.
- Шаг 3.7 (выполнен): перед подтверждением заказчик видит превью вакансии (заголовок и описание) в деталях заказа.
- Шаг 4.1 (выполнен): для соискателя исправлена видимость опубликованных вакансий в разделе заказов (`Биржа` показывает неоткликнутые, `Мои` — откликнутые).
- Шаг 4.2 (выполнен): в `workspace` подключен реальный отклик соискателя на вакансию (`POST /api/vacancies/{vacancyId}/respond`) из деталей заказа.
- Шаг 4.3 (выполнен): для исполнителя счётчик откликов по заказу синхронизирован с фактическими откликами кандидатов по vacancy.
- Шаг 4.4 (выполнен): для заказчика в `workspace` показаны только кандидаты стадии `Shortlist` (и выбранный кандидат), остальные стадии скрыты.
- Шаг 4.5 (выполнен): в деталях заказа исполнитель видит имена соискателей, которые откликнулись на связанную вакансию.
- Шаг 4.6 (выполнен): в деталях заказа исполнитель может открыть профиль откликнувшегося соискателя.
- Шаг 4.7 (выполнен): в деталях заказа исполнитель может переводить соискателя в `Shortlist` и возвращать в `Pool` (действие «Отказать» в текущем backend-контракте).
- Шаг 4.8 (выполнен): в действиях по соискателю добавлена UI-иконка чата (без бизнес-логики, подготовка к подключению чата).

# Candidate Resume MVP Rules

Дата фиксации: 2026-04-02
Статус: baseline требований (до реализации API)

## 1. Контекст процесса

1. `Customer` создает заказ на подбор.
2. Несколько `Executor` откликаются.
3. `Customer` выбирает одного `Executor`.
4. Выбранный `Executor` ведет вакансию и подбор кандидатов.

## 2. Роли и владение резюме

- `Executor` может создать резюме кандидата для заказчика только если кандидат отсутствует в системе.
- Если кандидат зарегистрирован в системе, резюме полностью в зоне ответственности кандидата.
- После регистрации кандидата управление резюме выполняет только кандидат.
- `Customer` не редактирует резюме, только просматривает переданные материалы и принимает решение.

## 3. Правила создания и изменения

- Запрещено создавать резюме от имени `Executor` для зарегистрированного кандидата.
- Для зарегистрированного кандидата `Executor` работает через привязку кандидата к вакансии, а не через создание дубля резюме.
- Для незарегистрированного кандидата `Executor` может создать карточку резюме для работы по конкретной вакансии.

## 4. Модель кандидатов в вакансии (2 списка + 1 выбор)

- В рамках одной вакансии есть рабочий список кандидатов (`pool`) — кандидаты, которых ведет `Executor`.
- Из рабочего списка `Executor` формирует список для заказчика (`shortlist`) и отправляет на выбор.
- `Customer` выбирает ровно одного кандидата только из `shortlist`; это не отдельный список, а одно поле выбранного кандидата у вакансии.

Инварианты:

- Кандидат может попасть в `shortlist` только если уже есть в `pool` этой вакансии.
- Выбранный `Customer` кандидат должен существовать в `shortlist` этой же вакансии.
- На вакансию допускается только один выбранный кандидат.

## 5. Что должно быть в форме резюме (зафиксировано из обсуждения)

- Не свободный текстовый блок, а структурированная форма с отдельными полями.
- Поле специализации через список/справочник (dropdown).
- Возможность прикрепить дополнительные материалы, включая видео-визитку и PDF.

## 6. Материалы, которые заказчик получает по кандидату

- Резюме кандидата.
- Комментарии и заметки по встречам.
- Ответы на вопросы в рамках отбора.

## 7. Ограничения и безопасность

- Выгрузка резюме в `PDF` разрешена только в обезличенном формате.
- В `PDF` обязательно исключаются прямые персональные идентификаторы: ФИО и контактные данные (телефон, email, мессенджеры, ссылки на соцсети).
- Полная защита от скриншотов технически не гарантируется; это фиксируется как ограничение.
